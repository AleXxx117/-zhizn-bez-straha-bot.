# Telegram-бот «Жизнь без страха»

Готовый бот психологической поддержки в стиле сообщества Саши. Он работает круглосуточно, отвечает на свободный текст и кнопки, проводит короткую практику, поддерживает ежедневные сообщения и показывает отдельный безопасный сценарий при угрозе жизни.

Боту не нужен платный ИИ или ключ OpenAI. В проекте нет сторонних Python-библиотек.

## Что умеет бот

- Поддержка при тревоге, панической волне, навязчивых мыслях, откате, агорафобии, дереализации, сканировании тела, бессоннице и одиночестве.
- Фирменные принципы: «пусть будет», позиция наблюдателя, меньше проверок, маленькими шагами возвращаться в жизнь.
- Практика на пять минут с четырьмя последовательными шагами.
- Добровольная ежедневная поддержка по времени Киева.
- Ссылки на сайт «Жизнь без страха», YouTube и Telegram-сообщество.
- Минимальная статистика для администратора без сохранения текста личных сообщений.
- Отдельная реакция на сообщения о самоповреждении и медицинские тревожные признаки.

Бот не ставит диагнозы, не назначает лекарства и не заменяет врача, психолога или экстренную службу.

## Важно: сначала замените опубликованный токен

Токен, который уже был отправлен сообщением, использовать нельзя.

1. Откройте `@BotFather`.
2. Отправьте `/revoke` и выберите своего бота.
3. Получите новый токен. Если BotFather не выдал его сразу, отправьте `/token`.
4. Никому не пересылайте новый токен и не добавляйте его в публичные файлы.

## Быстрый запуск на компьютере

Нужен Python 3.11 или новее.

1. Сделайте копию `.env.example` с именем `.env`.
2. Откройте `.env` и вставьте новый токен после `TELEGRAM_BOT_TOKEN=`.
3. В папке проекта выполните:

```bash
python3 bot.py
```

После строки `Бот @... запущен` откройте бота в Telegram и отправьте `/start`.

## Круглосуточный запуск через Docker

На сервере с установленным Docker:

```bash
cp .env.example .env
# Вставьте новый токен и нужные ссылки в .env
docker compose up -d --build
```

Проверка работы:

```bash
docker compose logs -f bot
```

Остановка:

```bash
docker compose down
```

## Запуск на Railway с телефона

1. Откройте [Railway](https://railway.com/new) и войдите через GitHub.
2. Нажмите `Deploy from GitHub repo` и выберите этот репозиторий.
3. Railway автоматически найдёт `Dockerfile` и соберёт бота.
4. Откройте сервис, затем вкладку `Variables` и добавьте:

```text
TELEGRAM_BOT_TOKEN=НОВЫЙ_ТОКЕН_ОТ_BOTFATHER
TIMEZONE=Europe/Kyiv
DAILY_HOUR=9
DAILY_MINUTE=0
DATA_DIR=/data
```

Токен вводится только в защищённой вкладке `Variables`. Не добавляйте его в GitHub, README, `.env.example` или сообщения.

5. Откройте `Deployments` → `View Logs`. Успешный запуск заканчивается строкой `Бот @... запущен`.
6. Вернитесь в Telegram, откройте своего бота и отправьте `/start`.

Для сохранения подписок и статистики между повторными развёртываниями можно добавить Railway Volume с путём подключения `/data`. Без Volume сам бот всё равно работает, но его локальная статистика может сбрасываться после новой сборки.

## Настройка администратора

1. Запустите бота и отправьте ему `/id`.
2. Скопируйте показанный номер в `.env` после `ADMIN_CHAT_ID=`.
3. Перезапустите бота.
4. Команда `/stats` станет доступна только этому ID.

## Добавление в группу

По умолчанию бот не вмешивается в каждый разговор. В группе он отвечает на команды, на упоминание `@имя_бота` и на ответы к его сообщениям. Для чувствительных тем предложите участникам писать боту лично.

Если нужно, чтобы Telegram передавал боту обычные групповые сообщения, настройте в `@BotFather` команду `/setprivacy`. Учитывайте приватность участников: бот всё равно не хранит текст сообщений, но участники должны понимать, что он их получает.

## Переменные настройки

| Переменная | Назначение |
| --- | --- |
| `TELEGRAM_BOT_TOKEN` | Новый секретный токен BotFather |
| `ADMIN_CHAT_ID` | ID Саши для команды `/stats` |
| `SITE_URL` | Ссылка на сайт |
| `YOUTUBE_URL` | Ссылка на YouTube-канал |
| `TELEGRAM_COMMUNITY_URL` | Ссылка на Telegram-сообщество |
| `TIMEZONE` | Часовой пояс, по умолчанию `Europe/Kyiv` |
| `DAILY_HOUR`, `DAILY_MINUTE` | Время ежедневной поддержки |
| `DATA_DIR` | Папка базы данных |
| `LOG_LEVEL` | Уровень журнала, по умолчанию `INFO` |

## Команды

- `/start` — приветствие и главное меню.
- `/support` — открыть поддержку.
- `/practice` — начать практику.
- `/daily` — включить или выключить ежедневные сообщения.
- `/help` — возможности и ограничения.
- `/privacy` — какие данные сохраняются.
- `/id` — показать ID чата.
- `/stats` — обезличенная статистика, только для администратора.
